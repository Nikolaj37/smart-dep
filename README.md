# Development

## Некоторые правила разработки:
- Для добавления какой-либо фичи добавляется ветка `feature/<issue_number>`, работа ведется в ней, после окончания тестирования и проверок делается PullRequest (PR) и после уже код вливается в develop;
- `master` ветка используется только для конечных релизов;
- `develop` ветка для стабильных версий системы;
- При необходимости внесения фиксов создается ветка `fix/<issue_number>`, аналогично остальным веткам;
